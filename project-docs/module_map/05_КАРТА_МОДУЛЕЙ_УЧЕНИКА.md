# 05. Карта модулей ученика

## Назначение документа

Этот документ помогает ученику видеть, из каких модулей состоит его личный проект и как они связаны.

## Когда обновлять

- после добавления нового модуля;
- после изменения связи между модулями;
- после важного этапа roadmap;
- после успешного run Codex, который меняет архитектуру.

## Как читать карту модулей

Сначала смотри на модуль, потом на его путь, потом на связи и проверки.

## Связи между модулями

- какой модуль получает вход;
- какой модуль обрабатывает данные;
- какой модуль сохраняет результат;
- какой модуль показывает результат в UI;
- какой модуль вызывает следующий модуль.

## Source/runtime границы

- source - это git-tracked код и документы;
- runtime - это живое состояние сервера;
- не смешивать их в одной строке и не править runtime как source.

## Что нельзя менять напрямую

- runtime файлы;
- секреты;
- БД без детерминированного инструмента;
- чужие agent packages;
- vendor checkout;
- systemd/nginx без отдельного proof.

## Последнее обновление

Заполняется вручную после очередного изменения.

## История обновлений

- дата;
- что изменилось;
- какой run это подтвердил.
