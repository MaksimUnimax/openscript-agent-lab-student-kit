# Prefix

## Что такое prefix

Prefix - это текст с правилами проекта.
Он вставляется перед сообщением ученика в ChatGPT.

Prefix нужен, чтобы ChatGPT каждый раз видел:
- актуальные правила работы;
- требование проверять ТЗ;
- требование не гадать;
- требование идти одним шагом;
- требование давать Codex prompt одним цельным блоком;
- требование проверять отчёты Codex.

Prefix можно редактировать.
Если проект меняется, prefix тоже можно обновлять.
Если расширение отключено, prefix можно скопировать вручную.

Важно:
prefix не должен содержать секреты, токены, пароли, приватные ключи или закрытые данные.

Здесь лежит текст prefix/reminder, который должен попасть в будущее расширение.

Для служебных prompt-кнопок см. папку `../prompt_buttons/`.

Файл:

- openscript_student_prefix.md

Перед генерацией расширения ChatGPT должен заменить placeholders на значения конкретного проекта ученика.

## Требование к форматированию prefix

Prefix должен попадать в расширение как многострочный plain text.

Нельзя склеивать prefix в одну строку.

Правильно:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.

USER_MESSAGE:

---
~~

Неправильно:

~~text
PROJECT_RULES_REMINDER:Перед ответом проверь актуальные правила проекта...
~~

При автоматической отправке ChatGPT итоговое сообщение должно начинаться с prefix.

Prefix нельзя вставлять после сообщения ученика.

Prefix нельзя вставлять в позицию курсора.

Правильная сборка:

~~text
prefix + "\n\n" + user_message
~~
