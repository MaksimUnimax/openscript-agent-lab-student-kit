# ТЗ: расширение ChatGPT для автоматической вставки project prefix

## 1. Цель расширения

Расширение нужно, чтобы перед каждым сообщением ученика в ChatGPT автоматически добавлялся рабочий prefix проекта.

Prefix — это текст с правилами работы проекта.

Он напоминает ChatGPT:

- проверять актуальные правила и ТЗ;
- не гадать;
- идти одним шагом;
- давать Codex prompt целиком;
- объяснять простым русским языком;
- не делать hardcode, костыли и обходы архитектуры.

Главная цель расширения:

ученик пишет обычное сообщение, нажимает штатную кнопку отправки ChatGPT, а расширение перед отправкой само превращает сообщение в:

~~text
PROJECT_RULES_REMINDER:
...

USER_MESSAGE:

---

<сообщение ученика>
~~

Ученик не должен каждый раз вручную нажимать кнопку вставки prefix.

Ручная кнопка вставки может быть в интерфейсе только как дополнительная проверочная функция.

Основной рабочий сценарий — автоматическая вставка prefix при штатной отправке сообщения в ChatGPT.

---

## 2. Главный пользовательский сценарий

Как должно работать:

1. Ученик открывает ChatGPT.
2. Ученик пишет обычный текст в поле сообщения, например:

~~text
Тест
~~

3. Ученик нажимает обычную кнопку отправки ChatGPT.
4. Расширение перехватывает отправку.
5. Расширение берёт текущий текст ученика.
6. Расширение проверяет, стоит ли `PROJECT_RULES_REMINDER:` в начале сообщения.
7. Если prefix ещё не стоит в начале, расширение заменяет весь текст поля на:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.
Не гадай. Сначала proof/current state, потом один следующий шаг.
Codex prompt давай только полный self-contained prompt целиком.
Решение должно быть универсальным: без hardcode, костылей и обхода архитектуры.
Объясняй простым русским языком.

USER_MESSAGE:

---

Тест
~~

8. После успешной вставки prefix расширение повторяет отправку сообщения.
9. В ChatGPT должно уйти сообщение, которое начинается с `PROJECT_RULES_REMINDER:`.

---

## 3. Что является правильным результатом

Если ученик написал:

~~text
Тест
~~

и нажал штатную кнопку отправки ChatGPT, в чат должно уйти:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.
Не гадай. Сначала proof/current state, потом один следующий шаг.
Codex prompt давай только полный self-contained prompt целиком.
Решение должно быть универсальным: без hardcode, костылей и обхода архитектуры.
Объясняй простым русским языком.

USER_MESSAGE:

---

Тест
~~

Важно:

- prefix стоит строго в начале сообщения;
- после `PROJECT_RULES_REMINDER:` есть перенос строки;
- строки prefix не склеены;
- `USER_MESSAGE:` отделён пустыми строками;
- сообщение ученика идёт после блока `---`.

---

## 4. Что запрещено

Запрещён результат:

~~text
ТестPROJECT_RULES_REMINDER:
...
~~

Запрещён результат:

~~text
Тест

PROJECT_RULES_REMINDER:
...
~~

Запрещён результат:

~~text
PROJECT_RULES_REMINDER:Перед ответом проверь...
~~

Запрещено:

- склеивать строки prefix в одну строку;
- терять пустые строки между блоками;
- вставлять prefix в позицию курсора;
- добавлять prefix в конец сообщения;
- отправлять сообщение, где пользовательский текст стоит перед prefix;
- добавлять prefix повторно, если он уже стоит в начале сообщения;
- использовать clipboard/буфер обмена для автоматической отправки сообщения;
- считать ручную кнопку “Вставить сейчас” основным рабочим сценарием;
- считать “можно скопировать prefix вручную” готовым решением.

Буфер можно использовать только для отдельных кнопок:

- `Копировать prompt обновления документов`;
- `Копировать prompt старта нового диалога`;
- `Копировать prefix`.

Кнопка автоматической отправки через ChatGPT не должна менять clipboard.

---

## 5. Поведение при штатной кнопке отправки ChatGPT

Расширение должно слушать нажатие штатной кнопки отправки ChatGPT.

Под штатной кнопкой отправки понимается обычная кнопка ChatGPT, которой пользователь отправляет сообщение.

При клике по этой кнопке расширение должно:

1. проверить, включено ли расширение;
2. проверить, включена ли опция автоматической вставки prefix перед отправкой;
3. найти поле ввода ChatGPT;
4. получить текущий текст пользователя;
5. если текст пустой — не вмешиваться;
6. если текст уже начинается с `PROJECT_RULES_REMINDER:` — не менять его и разрешить отправку;
7. если текст не начинается с `PROJECT_RULES_REMINDER:`:
   - остановить текущую отправку;
   - заменить содержимое поля на `prefix + "\n\n" + user message`;
   - проверить, что поле теперь начинается с `PROJECT_RULES_REMINDER:`;
   - дождаться обновления состояния поля;
   - повторить отправку.

Расширение не должно отправлять сообщение без действия пользователя.

Автоотправка допускается только как продолжение пользовательского действия:

1. пользователь нажал штатную кнопку отправки ChatGPT;
2. расширение остановило первый клик;
3. расширение добавило prefix;
4. расширение повторило отправку.

---

## 6. Поведение при Enter

Если ChatGPT отправляет сообщение по Enter, расширение должно работать так же, как при клике по кнопке отправки:

1. перехватить Enter;
2. поставить prefix в начало;
3. повторить отправку.

Если пользователь нажал Shift+Enter, это перенос строки.

В этом случае отправку перехватывать не нужно.

## 7. Формат prefix

Базовый prefix:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.
Не гадай. Сначала proof/current state, потом один следующий шаг.
Codex prompt давай только полный self-contained prompt целиком.
Решение должно быть универсальным: без hardcode, костылей и обхода архитектуры.
Объясняй простым русским языком.

USER_MESSAGE:

---
~~

После prefix должны идти две новые строки и затем сообщение ученика.

Итоговая сборка сообщения:

~~text
<prefix>

<user_message>
~~

Пример:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.
Не гадай. Сначала proof/current state, потом один следующий шаг.
Codex prompt давай только полный self-contained prompt целиком.
Решение должно быть универсальным: без hardcode, костылей и обхода архитектуры.
Объясняй простым русским языком.

USER_MESSAGE:

---

Тест
~~

## 8. Сохранение переносов строк

Расширение обязано сохранять переносы строк prefix.

Нельзя превращать prefix в одну строку.

Плохо:

~~text
PROJECT_RULES_REMINDER:Перед ответом проверь актуальные правила проекта...
~~

Хорошо:

~~text
PROJECT_RULES_REMINDER:
Перед ответом проверь актуальные правила проекта и текущий контекст.
Если правила недоступны или не прочитаны — STOP, не выдавай prompt/fix.
~~

Для contenteditable-поля ChatGPT нужно вставлять текст так, чтобы визуально и фактически сохранялись переносы строк.

Допустимые варианты реализации:

1. корректная вставка plain text через browser editing API;
2. замена содержимого composer на HTML, где абзацы и переносы соответствуют исходному plain text;
3. другой способ, который даёт в отправленном сообщении правильный многострочный текст.

Недопустимый результат — склеенный prefix без переносов.

---

## 9. Поведение с уже вставленным prefix

Если сообщение уже начинается с:

~~text
PROJECT_RULES_REMINDER:
~~

расширение не должно добавлять prefix повторно.

Если сообщение содержит `PROJECT_RULES_REMINDER:` не в начале, это считается испорченным состоянием после старой ошибки.

В этом случае расширение должно привести сообщение к нормальному виду:

~~text
PROJECT_RULES_REMINDER:
...

USER_MESSAGE:

---

<текст ученика>
~~

Нельзя отправлять сообщение, где пользовательский текст стоит перед prefix.

## 10. Интерфейс расширения

В popup расширения должны быть элементы.

### 10.1. Переключатель включения

Название:

~~text
Включено
~~

Если выключено — расширение ничего не вставляет перед отправкой.

### 10.2. Переключатель автоподстановки

Название:

~~text
Вставлять prefix при нажатии “Отправить” в ChatGPT
~~

Если выключено — штатная кнопка отправки ChatGPT работает без prefix.

### 10.3. Поле редактирования prefix

Пользователь должен видеть prefix и иметь возможность его изменить.

Изменённый prefix сохраняется локально в браузере.

### 10.4. Кнопка “Сохранить”

Сохраняет текущие настройки и prefix.

### 10.5. Кнопка “Вставить prefix перед текстом”

Ручная вспомогательная кнопка.

Она должна взять текущий текст в поле ChatGPT и заменить его на:

~~text
prefix + "\n\n" + current_user_message
~~

Эта кнопка нужна только для ручной проверки.

Основной сценарий — автоматическая вставка при штатной отправке ChatGPT.

### 10.6. Кнопка “Копировать prompt обновления документов”

Копирует в буфер prompt из документа:

~~text
extension/prompt_buttons/docs_update_prompt.md
~~

или из встроенного файла расширения с таким же смыслом.

### 10.7. Кнопка “Копировать prompt старта нового диалога”

Копирует в буфер prompt из документа:

~~text
extension/prompt_buttons/new_dialogue_start_prompt.md
~~

или из встроенного файла расширения с таким же смыслом.

### 10.8. Кнопка “Копировать prefix”

Копирует prefix в буфер.

### 10.9. Кнопка “Сбросить”

Возвращает стандартный prefix.

### 10.10. Блок диагностики

В popup должен быть блок статуса/диагностики.

Минимальные поля:

~~text
extension_version:
active_tab_url:
is_chatgpt_url:
composer_found:
composer_selector:
current_text_length:
current_has_prefix:
current_starts_with_prefix:
prefix_length:
prefix_has_marker:
autoBeforeSend:
enabled:
last_action:
last_result:
~~

## 11. Prompt-кнопки

Расширение должно иметь две prompt-кнопки:

~~text
Копировать prompt обновления документов
Копировать prompt старта нового диалога
~~

Эти кнопки только копируют текст в буфер.

Они не должны:

- вставлять prompt в ChatGPT автоматически;
- отправлять prompt автоматически;
- менять поле ввода ChatGPT;
- смешиваться с prefix-вставкой.

Тексты prompt’ов брать из:

~~text
extension/prompt_buttons/docs_update_prompt.md
extension/prompt_buttons/new_dialogue_start_prompt.md
~~

Папку `extension/prompt_buttons/` нельзя удалять, переписывать или заменять при генерации расширения.

---

## 12. Данные проекта ученика

Prompt’ы для кнопок не должны содержать чужие repo URL, чужие SSH URL, чужие пути сервера или чужие имена проектов.

В prompt’ах должны использоваться placeholders:

~~text
<STUDENT_PROJECT_NAME>
<STUDENT_PUBLIC_REPO_URL>
<STUDENT_WORKING_REPO_URL>
<STUDENT_WORKING_REPO_SSH_URL>
<STUDENT_SERVER_PROJECT_ROOT>
<STUDENT_DOCS_ROOT>
<STUDENT_PUBLIC_DOCS_SOURCE>
<STUDENT_PRIVATE_SOURCE_REPO>
<STUDENT_PUBLIC_DOCS_REPO>
~~

Запрещён hardcode:

~~text
MaksimUnimax
openscript-agent-lab-docs
/opt/openscript-agent-lab
git@github.com:MaksimUnimax/*
https://github.com/MaksimUnimax/*
~~

---

## 13. Хранение данных

Расширение хранит настройки локально в браузере.

Допустимо использовать:

~~text
chrome.storage.local
~~

Расширение не должно отправлять данные наружу.

Расширение не должно иметь backend.

Расширение не должно читать секреты.

Расширение не должно требовать токены, API keys или авторизацию.

---

## 14. Безопасность

Расширение не должно:

- читать cookies;
- читать токены;
- читать localStorage ChatGPT;
- отправлять сетевые запросы наружу;
- отправлять данные на сервер;
- менять содержимое страницы, кроме поля ввода ChatGPT;
- отправлять сообщение без действия пользователя.

Буфер обмена не должен использоваться в основном сценарии автоотправки.

---

## 15. Техническая логика вставки

### 15.1. Найти поле ввода

Расширение должно уметь находить composer ChatGPT.

Проверяемые selectors:

~~text
[data-testid="prompt-textarea"]
#prompt-textarea
textarea
.ProseMirror
div[contenteditable="true"]
[contenteditable="true"]
~~

Если найдено несколько кандидатов, брать видимый и актуальный composer.

### 15.2. Получить текущий текст

Нужно получить plain text текущего сообщения ученика.

Для textarea/input:

~~text
element.value
~~

Для contenteditable:

~~text
element.innerText
~~

### 15.3. Собрать итоговый текст

~~text
finalText = prefix.trim() + "\n\n" + userText.trim()
~~

Если userText пустой, отправку лучше не трогать.

### 15.4. Заменить весь текст composer

Важно: не вставить prefix в курсор, а заменить весь текст composer.

Плохо:

~~text
userText + prefix
~~

Правильно:

~~text
prefix + userText
~~

### 15.5. Сообщить ChatGPT, что поле изменилось

После замены текста нужно вызвать события:

~~text
input
change
keyup
~~

или эквивалентный механизм, чтобы React/ChatGPT увидел новое состояние поля.

### 15.6. Повторить отправку

После успешной замены текста расширение должно повторить отправку.

Нужна небольшая задержка, чтобы ChatGPT успел обновить состояние поля.

Рекомендуемый диапазон:

~~text
150–400 ms
~~

---

## 16. Что считать успешной вставкой

После вставки текущий текст composer должен начинаться с:

~~text
PROJECT_RULES_REMINDER:
~~

Диагностика должна показать:

~~text
current_starts_with_prefix: yes
prefix_has_marker: yes
~~

Если `current_has_prefix: yes`, но `current_starts_with_prefix: no`, это ошибка.

Такой результат нельзя отправлять.

## 17. Acceptance tests

### Test 1. Простая отправка

Ввод:

~~text
Тест
~~

Действие:

~~text
Нажать штатную кнопку отправки ChatGPT
~~

Ожидаемый результат:

~~text
PROJECT_RULES_REMINDER:
...
USER_MESSAGE:

---

Тест
~~

Сообщение должно начинаться с `PROJECT_RULES_REMINDER:`.

### Test 2. Нельзя склеивать строки

Ожидаемый результат не должен быть таким:

~~text
PROJECT_RULES_REMINDER:Перед ответом проверь...
~~

Между `PROJECT_RULES_REMINDER:` и следующим текстом должен быть перенос строки.

### Test 3. Prefix не должен вставляться после текста

Ввод:

~~text
Тест
~~

Запрещённый результат:

~~text
ТестPROJECT_RULES_REMINDER:
...
~~

### Test 4. Повторная отправка

Если в поле уже есть:

~~text
PROJECT_RULES_REMINDER:
...
~~

расширение не должно добавлять второй prefix.

### Test 5. Prompt-кнопки

Кнопка:

~~text
Копировать prompt обновления документов
~~

должна копировать prompt обновления документов.

Кнопка:

~~text
Копировать prompt старта нового диалога
~~

должна копировать prompt старта нового диалога.

Обе кнопки не должны отправлять prompt автоматически.

### Test 6. Буфер не используется для автоотправки

При обычной отправке ChatGPT расширение не должно менять clipboard.

Clipboard может меняться только при нажатии кнопок:

~~text
Копировать prompt обновления документов
Копировать prompt старта нового диалога
Копировать prefix
~~

### Test 7. Диагностика

После нажатия “Проверить” должно быть видно:

~~text
composer_found: yes
prefix_has_marker: yes
autoBeforeSend: yes
enabled: yes
~~

---

## 18. Что не является решением

Не является решением:

- кнопка “Вставить сейчас” как основной способ работы;
- копирование prefix в буфер перед отправкой;
- вставка prefix после текста пользователя;
- склеивание prefix в одну строку;
- ручная инструкция “нажми Ctrl+V”;
- повторяющиеся prefix;
- отправка сообщения без prefix;
- отправка сообщения, где prefix есть, но не в начале.

---

## 19. Главный критерий готовности

Расширение считается готовым только если ученик может работать так:

1. открыть ChatGPT;
2. написать обычное сообщение;
3. нажать обычную кнопку отправки ChatGPT;
4. получить отправленное сообщение, которое начинается с `PROJECT_RULES_REMINDER:`;
5. не нажимать отдельную кнопку вставки prefix;
6. не использовать Ctrl+V;
7. не получать сломанный формат prefix;
8. иметь в popup две prompt-кнопки.

Главная проверка:

~~text
Ввод:
Тест

Действие:
Обычная кнопка отправки ChatGPT

Итог:
PROJECT_RULES_REMINDER:
...
USER_MESSAGE:

---

Тест
~~

---

## 20. Отдельно: что нельзя трогать при генерации расширения

При генерации zip расширения нельзя удалять или переписывать документацию prompt-кнопок:

~~text
extension/prompt_buttons/README.md
extension/prompt_buttons/docs_update_prompt.md
extension/prompt_buttons/new_dialogue_start_prompt.md
~~

Эти файлы являются источником текстов для двух кнопок.

Если в этих файлах есть ошибки placeholders, это исправляется отдельным docs-run, а не при генерации zip расширения.
