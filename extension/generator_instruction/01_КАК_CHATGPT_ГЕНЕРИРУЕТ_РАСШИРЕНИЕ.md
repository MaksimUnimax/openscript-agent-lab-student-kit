# Как ChatGPT генерирует расширение

Этот документ предназначен для ChatGPT.

## Главный документ для генерации

Перед генерацией расширения ChatGPT обязан прочитать каноническое ТЗ:

- `extension/generator_instruction/02_ТЗ_РАСШИРЕНИЕ_CHATGPT_PREFIX.md`

Если в этом документе есть более строгие требования, чем в старой инструкции, главным считается ТЗ.

Запрещено генерировать zip расширения, не проверив требования из ТЗ.

## Назначение

Расширение нужно, чтобы вставлять в начало сообщения специальный prefix/reminder с правилами проекта.

Prefix находится здесь:

extension/prefix/openscript_student_prefix.md

## Что объяснить ученику перед генерацией

Перед генерацией zip ChatGPT должен объяснить ученику простыми словами:

Расширение - это маленький инструмент для браузера.
Оно вставляет в сообщение ChatGPT правила работы проекта.
Это нужно, чтобы ChatGPT не забывал методику: читать документы, не гадать, идти одним шагом и давать Codex prompt правильным форматом.

Ученик должен понимать:
- расширение не программирует проект;
- расширение только помогает вставлять правила;
- правила можно редактировать;
- расширение можно отключить;
- главный обязательный сценарий:
  1. ученик пишет обычное сообщение в ChatGPT;
  2. ученик нажимает штатную кнопку отправки ChatGPT;
  3. расширение перед отправкой ставит prefix строго в начало сообщения;
  4. сообщение уходит уже с prefix;
- ручная кнопка “Вставить prefix” может быть только дополнительной проверочной функцией;
- она не является основным способом работы;
- ручное копирование prefix - только аварийный вариант, а не критерий готовности расширения;
- данные не должны отправляться на внешний сервер.

## Важное правило

В учебном комплекте не хранится готовый zip расширения.

ChatGPT должен сгенерировать расширение позже, прямо в диалоге с учеником, когда ученик дойдёт до этапа установки расширения.

После генерации ChatGPT даст ученику отдельную простую инструкцию:
- как скачать zip;
- как распаковать;
- как открыть страницу расширений браузера;
- как включить режим разработчика;
- как установить расширение;
- как проверить, что prefix вставляется.

Эту пользовательскую инструкцию не нужно заранее хранить здесь как финальный шаг.

## Что должно уметь расширение

Минимальная версия расширения должна:

1. хранить текст prefix/reminder;
2. давать возможность включить или выключить вставку prefix;
3. давать поле для редактирования prefix;
4. иметь кнопку сохранения;
5. иметь кнопку копирования prefix;
6. вставлять prefix в поле сообщения ChatGPT при нажатии штатной кнопки отправки ChatGPT;
7. не хранить секреты;
8. не отправлять данные наружу;
9. работать локально в браузере ученика.

## Минимальные возможности расширения

Расширение должно уметь:

1. хранить prefix локально в браузере;
2. показывать prefix в поле редактирования;
3. сохранять изменения prefix;
4. включать и выключать автоматическую вставку;
5. копировать prefix одной кнопкой;
6. вставлять prefix в поле сообщения ChatGPT при нажатии штатной кнопки отправки ChatGPT;
7. не отправлять данные наружу;
8. не хранить секреты;
9. не требовать серверной части.

## Кнопки служебных prompt'ов

При генерации расширения ChatGPT должен добавить в интерфейс две кнопки:

1. “Копировать prompt обновления документов”
2. “Копировать prompt старта нового диалога”

Кнопки должны быть размещены рядом с основными действиями расширения, чтобы ученик быстро находил их.

Поведение кнопок:

- кнопка “Копировать prompt обновления документов” копирует текст из `extension/prompt_buttons/docs_update_prompt.md`;
- кнопка “Копировать prompt старта нового диалога” копирует текст из `extension/prompt_buttons/new_dialogue_start_prompt.md`;
- перед генерацией расширения ChatGPT должен заменить placeholders на данные проекта ученика;
- если данных не хватает, ChatGPT должен STOP и запросить недостающие значения;
- prompt’ы не должны содержать чужие repo URLs, SSH URLs, server paths или project names;
- prompt’ы не должны отправляться автоматически.

Минимальная проверка:

- обе кнопки видны в popup;
- каждая кнопка копирует свой prompt в буфер;
- в скопированном тексте нет чужих repo URL;
- placeholders заменены или ChatGPT честно остановился до генерации.

## Acceptance test перед выдачей zip ученику

Перед тем как отдать ученику zip расширения, ChatGPT должен проверить будущую сборку по смыслу ТЗ.

Минимальный тест:

Ввод ученика:

~~text
Тест
~~

Действие ученика:

~~text
Нажать обычную кнопку отправки ChatGPT
~~

Ожидаемый результат:

~~text
PROJECT_RULES_REMINDER:
...
USER_MESSAGE:

---

Тест
~~

Fail-результаты:

~~text
ТестPROJECT_RULES_REMINDER:
~~

~~text
PROJECT_RULES_REMINDER:Перед ответом проверь...
~~

Если получается один из fail-результатов, zip нельзя считать готовым.

## Что ChatGPT должен подставить в prefix

Перед генерацией расширения ChatGPT должен заменить placeholders:

```text
<PROJECT_NAME>
<DOCS_SOURCE>
<STUDENT_PROJECT_REPO>
<PROJECT_RULES_PATH>
<PROJECT_TZ_PATH>
```

Если какие-то значения неизвестны, ChatGPT должен STOP и запросить недостающую информацию.

## Что нельзя делать

Нельзя:
- вставлять секреты;
- вставлять токены;
- вставлять private keys;
- hardcode IP тестового сервера;
- hardcode чужой проект;
- делать расширение зависимым от одного конкретного домена, если это не согласовано;
- добавлять отправку prefix на внешний сервер;
- делать расширение обязательным для работы проекта, если ученик ещё не дошёл до этого шага.

## Результат

Результатом будущего шага должна быть zip-архивация расширения, созданная ChatGPT в текущем диалоге.

## Что считается успешной установкой

Этап расширения принят только если ученик подтвердил:

1. zip расширения создан ChatGPT;
2. расширение установлено в браузер;
3. prefix виден в расширении;
4. prefix можно редактировать;
5. расширение можно выключить;
6. prefix автоматически вставляется в начало сообщения при нажатии штатной кнопки отправки ChatGPT;
7. отправленное сообщение начинается с `PROJECT_RULES_REMINDER:`;
8. prefix не склеен в одну строку;
9. prompt-кнопки видны и копируют свои prompt’ы;
10. ручное копирование prefix не считается основным успешным сценарием;
11. ученик понимает, зачем расширение нужно.
