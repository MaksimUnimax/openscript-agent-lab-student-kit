# 04. Правило одного шага

Один Codex run = одна задача.

## Нельзя смешивать в одном шаге

- сервер;
- git;
- docs;
- Hermes;
- UI;
- Telegram;
- tool registry;
- финансовый инструмент;
- БД;
- фото чека;
- голос;
- второй агент.

## Как работать

1. Перед fix сначала нужен proof текущего состояния.
2. Если задача большая, её нужно разбить на маленькие шаги.
3. После каждого шага нужен отчёт и проверка.
4. Не переходить дальше, если предыдущий шаг не принят.

## Run counter

- ChatGPT ведёт счётчик рабочих run/шагов.
- По умолчанию каждые 10 run нужно проверить, не пора ли обновить документы.
- После крупного этапа документы можно обновлять раньше.
- Если были только проверки без изменений, можно объяснить, почему docs update откладывается.

## Какие docs обновлять

- roadmap;
- ТЗ;
- карта модулей;
- актуальный снимок;
- контекст диалога ученика;
- current status, если он есть.
