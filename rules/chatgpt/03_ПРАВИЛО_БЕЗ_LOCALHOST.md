# 03. Правило без localhost

Это жёсткое правило для всех учебных шагов.

## Запрещено давать ученику

1. `localhost`
2. `127.0.0.1`
3. адрес локальной машины
4. IP тестового сервера другого человека

## Разрешённые шаблоны

- `http://<SERVER_PUBLIC_IP>/<PATH>`
- `http://<SERVER_PUBLIC_IP>:<PORT>/<PATH>`
- `https://<STUDENT_DOMAIN>/<PATH>`

## Для страницы структуры проекта

- `http://<SERVER_PUBLIC_IP>/project-structure/`
- `http://<SERVER_PUBLIC_IP>:<PORT>/project-structure/`

## Правило проверки

1. Если публичный IP или домен не доказан, нужно STOP и сначала proof.
2. Если Codex дал `localhost` как адрес для ученика, шаг не принят.
3. `localhost` допустим только как внутренняя техническая проверка Codex, но не как адрес для ученика.
4. В документах использовать только шаблоны, не hardcode IP тестового сервера.
